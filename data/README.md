# Data Directory

This directory stores lightweight data instructions and documents the local-only artifacts generated by the project pipeline.

## Directory Purpose

The repository should only include small, shareable, and reproducible data-related assets.

## Expected Structure

- `raw/`: local-only downloaded assets such as the Hart labels file
- `interim/`: local-only acquisition outputs
- `processed/`: local-only cleaned dataset and reproducible split CSVs
- `sample/`: optional small examples that are safe to version

## Current Protocol

The recommended data flow is:

1. acquisition writes `data/interim/merged_filtered.csv`
2. preprocessing writes `data/processed/merged_filtered_clean.csv`
3. preprocessing also writes:
   - `data/processed/splits/train.csv`
   - `data/processed/splits/validation.csv`
   - `data/processed/splits/test.csv`
   - `data/processed/splits/metadata.json`

The split metadata file is the reproducibility anchor for the final training and evaluation protocol.

## Versioning Policy

Do not commit:

- large image datasets
- raw Kaggle downloads
- generated heavy CSV files
- temporary preprocessing outputs
- model checkpoints stored together with data

## Reproducibility Note

All large data assets should be recreated through the documented acquisition and preprocessing scripts rather than stored directly in Git.
